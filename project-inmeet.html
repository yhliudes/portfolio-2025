01.
Project
Overview
專案總覽

Inmeet：讓交友更輕鬆的互動平台

Inmeet 是一款充滿活力的交友應用，讓你輕鬆自在地認識新朋友！我們專注於簡設計，摒棄冗餘功能，讓你在首頁快速找到心儀對象，隨時展開聊天，享受無壓力的交友體驗！

專案目標

我們希望藉由此 app 讓設計概念相似度已經過高的交友軟體 app 帶入新的概念，同時讓交友更加快速便利以及安全。

市場現況

目前市面上的交友軟體絕大多數都是相似於 tinder 的往右滑概念，各款交友軟體的相似度過高，我們為了與市場做出區隔，希望藉由 inmeet 讓交友軟體更加直覺，快速速配到最適合的配對。

角色產出

在此專案中，我與客戶及工程師積極溝通與合作。我負責 UI 設計流程與產出，來回與客戶討論增減 app 應有功能以及視覺設計，最終交付給工程師。

04.

System
設計系統

在安排設計系統時，首先確定的是主視覺必須是桃紅色以及暗色背景系統，因此在設計完 icon 後，決定 app 整體使用黑色背景，配合上主色粉色以及輔助色，按鈕也以圓弧為主。

▲ inmeet 設計系統：主色調、輔助色、按鈕樣式

  • 主色調：桃紅色，營造活潑友善的氛圍
  • 背景系統：暗色系，突顯內容並減少視覺疲勞
  • 按鈕設計：圓弧為主，柔和親切
  • 視覺層次：清晰明確的資訊層級
05.
MVP
Features
最小可行產品 (MVP)

在確定此 app 核心功能後，我開始進行設計及流程細節，此 app 最必要的功能為配對以及聊天系統，我花了三天的時間製作核心的幾項功能，分別為隨機配對、篩選、約會頁、以及聊天頁。

▲ MVP 核心功能：配對、篩選、約會、聊天

核心功能優先順序

  1. 隨機配對：快速瀏覽使用者，找到心儀對象
  2. 篩選功能：依照條件精準搜尋
  3. 約會頁面：查看配對成功的使用者
  4. 聊天系統：即時通訊，建立連結
06.
Interface
Evolution
介面更迭

在快速製作出第一版視覺稿後，我們認為向左滑、向右滑的方式太過普遍，因此將配對方式修改為卡片條列式，讓使用者可以更一目瞭然。

▲ 介面演進：從滑動式改為卡片條列式

設計思考

捨棄傳統「左滑右滑」的配對方式，改為卡片條列式設計，讓使用者能夠同時瀏覽多位候選人，提供更直覺、更有效率的配對體驗。此設計也降低了誤觸的機率，讓使用者能更謹慎地選擇心儀對象。

07.
Login
Rewards
登入好禮：如何讓使用者黏著度提高

在將視覺設計完成到一階段後，我們轉為研究如何讓使用者黏著度提高，討論過後決定加入「登入好禮」以及「充足資訊」，讓使用者更願意提供自己的資訊以及樂於登入。

登入好禮新增重點

  • 註冊好禮及登入好禮：註冊即可享有 30 顆愛心，如每日註冊可再獲得額外愛心。
  • 資料完整度鼓勵：在註冊時需要填寫個人檔案，為了提升填寫意願，如在註冊時略過的資料，會在進入首頁時鼓勵填寫。
  • 持續提醒機制：如未填寫會在個人頁顯示，隨時可以再進行資料補填。

▲ 登入好禮機制：註冊獎勵與資料完整度提升策略

💎 提升黏著度的策略

  • 透過獎勵機制鼓勵每日登入
  • 漸進式引導填寫個人資料
  • 清楚顯示資料完整度，激勵補填
  • 降低新用戶的使用門檻

08.
Backend
System
後台系統

在設計 app 的同時，也進行後台系統的建置，分別為總後台、經銷商後台、總機後台，三個後台共約 80 個頁面並且持續增加當中。在製作後台系統時，由於需要考慮到與 app 配合的功能性，做了許多取捨與修改，也與客戶以及工程師積極溝通修改。

▲ 後台系統：總後台、經銷商後台、總機後台

後台系統架構

  • 總後台：整體營運數據、使用者管理、系統設定
  • 經銷商後台：業績追蹤、客戶管理、報表系統
  • 總機後台：客服聊天、問題處理、即時回應
  • 頁面總數：約 80 個頁面，持續擴充中

💡 設計挑戰：在製作後台系統時，需要考慮與 app 功能的串接與同步，並在功能性與易用性之間取得平衡。透過與客戶及工程師的持續溝通，逐步優化後台的操作流程與介面設計。

09.
Reflection
反思與感想

在此專案中我一人負責了介面設計、原型製作、設計系統、後台系統，從 0 到 1 的過程學到了非常多，也有許多需要改善與反思的地方：

＃設計脈絡需有邏輯

在製作 app 及後台途中與客戶討論做了非常多次的設計改版，我發現在一人設計時容易會有盲點，在這個時候必須跳脫框架以使用者的模式去思考，如何設計才是有邏輯並且對使用者方便的步驟。

＃與工程師的溝通重要性

有許多頁面在設計時，工程師並不了解或是無法做到的功能，在這種時候必須與工程師討論折衷的作法，或是需要說服工程師此設計的功能性以及正確性。例如在製作後台聊天室時，由於聊天室是由總機代替技師與會員聊天，此功能較為特殊，因此來回與工程師進行了許多討論，最後才決定了設計以及同時兼顧工程師能夠做到的聊天頁。

核心學習

  • 獨立設計的挑戰：學會跳脫自我視角，以使用者思維設計
  • 跨職能溝通：與工程師、客戶的有效協作
  • 快速迭代能力：在時間壓力下產出高品質設計
  • 系統性思考：同時考量前後台的整合性
